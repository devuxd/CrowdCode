Write test tutorial

In Write a Test, your goal is to implement a test case.

Given a description of a test case and function to be tested,

your task is to implement the test case, providing concrete values for the specified scenario.

Each value is specified as either a primitive or a JSON literal describing a data structure value. Pretty easy, right?

JSON literals can become long and complex:

But CrowdCode provides examples of each data type.

You can use Paste Example to insert a default value or browse the Available Data Types panel to copy a specific example.

If you discover an issue with either the function description or the test case, you should report it.
