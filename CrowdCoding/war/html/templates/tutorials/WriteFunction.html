Write function tutorial

In this microtask you are asked to edit the code of a function.
Pseudocall & pseudocode

When you implement a function from scratch (or editing a pre-written function), don’t do all the work alone!

You can sketch out part of the work without worrying about every detail by using pseudocode and pseudocalls.

Pseudocode

The pseudocode ( //# syntax) allows you to sketch algorithms or partial solutions and let the other workers know that there is some more work to do on this function.

Pseudocall

As a difference from the way you usually write code, in CrowdCode you don’t know which other functions are already implemented in the system. So if you want to call a function, you have to use the pseudocalls (//! syntax).

Pseudocalls are extremely useful when the code you have to write is complex and can’t be written in just a few minutes. They allow you to split the function editing with the other crowd-coders (and reuse existing functions).

Remember:

try to keep it simple and delegate the work if there is too much to do!
