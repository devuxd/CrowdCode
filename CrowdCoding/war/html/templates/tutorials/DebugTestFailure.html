Debug a test failure

In Debug a Test Failure, your goal is to find and correct any bug(s) that caused a function to fail one of its unit tests.

CrowdCode provides a test runner listing the results of executing each of the functionâ€™s tests. Passing tests are shown in green; failing tests in red.

Each test result lists the inputs and outputs for the test, as well as information about errors (if any).

For failed tests, the default view shows you the diff between the expected and the actual return value. Green background is the the expected field value and red background is the actual field value.
The fields that doesn't have a background color, have the same value.

If you are confused by the diff view, you can always switch to the normal one.

Debug Test Failure provides an editor for the function, enabling you to find bugs, implement a fix, and rerun the tests to check if it works.

What happens if the bug is not in the function itself but is in a function that it calls? For each test, CrowdCode lists all of the functions directly invoked by the function under test, describing each distinct set of inputs and the output value produced.

For example, here a function sum is being called with the parameters 1 and 2 and is returning the output of 4. This does not seem to match its specified behavior. To solve this problem, you can edit the return value, replacing 4 with 3. This creates a stub. When you rerun the tests, CrowdCode will use the return value specified by the stub rather than the actual return value, executing the function call as if it had returned 3. After submitting the Debug microtask, each stub will be translated into a test for the called function, expressing the desired behavior

If you need additional information about the behavior of a function, you can use the debug.log(String) function.

Results from debug.log() are shown for each test.

After doing changes to the function code or to the stubs, re-run the tests to see the changes

It might also be that the function itself is correct and it is the test which is incorrect. In this case, you should report an issue with the test.
