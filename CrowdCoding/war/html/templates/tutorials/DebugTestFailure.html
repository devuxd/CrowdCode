Debug a test failure tutorial

In this microtask, you’re asked to find and correct the bug in a function that didn’t pass the tests.

You have access to all the function’s test results

A test result is green and marked with a checkmark in the upper right corner of the test if the test has passed, and red with an x in the upper right corner if it failed.

A test result shows you the inputs and output used for that particular test, information about the errors (if any) and also the list of all the functions called by this one, with their input/output values.

These are called stubs and allows you to change the behavior of a called function for a specified set of inputs. In this way you can focus only on debugging a single function without worrying about the code of the other functions.

Finally, you can fix code in the function directly.

If you need to monitor a statement, use the function debug.log(statement).

If you re-run the tests, the log will be printed in the test result.

After doing changes to the function code or to the stubs, re-run the tests to see the changes

Finally, if you notice an issue with a test case, please report it
