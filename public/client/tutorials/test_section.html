Create a test

To create a new unit test, click “Add a new test”.

You can write a brief description of the purpose of the unit test and select either an Input/Output test or an Assertion test.

Input/Output tests

Input/output tests specify the desired behavior of the function providing an object definition for all the inputs and for the output.

Assertion tests

Assertion tests afford more flexibility, letting you check only parts of the output (e.g., if an object has a specific property), using function calls to check properties of the output (e.g., length of an array), using complex expressions to construct input values, and even using multiple asserts.

Running and debugging tests

Click “Run” to run all of the function’s tests and view the list of passing and failing tests.

Test details

Click on a listed test to see the test code and its result.

Inspecting the code

Step 1: In "inspect mode", clicking on inspect code

Step 2: In the function editor where you can hover over expressions to see an expression value popup. After editing the function, click “Run” and then again click “Inspect” to see the updated values.

Stub a function call

Step 1: In "inspect mode", clicking on inspect code

Step 2: Clicking on a function call opens a popup that allows to stub a function.

Step 3: In the expression value popup, click on “stub this function call” to automatically replace a function call for a specific input with a stub for an alternative specific output, and run the tests again with the stub for the invoked function.

Reporting an issue

If a test itself is incorrect, click “Report an issue” in the test detail page to report the issue to be addressed, temporarily disabling the test.
