Review

In Review Work, your goal is to assess work submitted by the crowd.You should read function codes and test, after that accept or reject the implementation
review tasks (reviewing code of the other programmer)

Step 1: review implementation of the behavior which is written by other programmer. + sign shows these lines are added. - sign shows these lines are deleted from the code.

Step 2: review tests of this behavior

Step 3: rate the contribution and give a feedback to the development

Step 4: once you’re finished, click “Submit” button

If you rate work with 1 to 3 stars, the work will be marked as needing revision. In this case, you must describe aspects of the work that you feel must be improved.

If you feel that the work as submitted is already of high quality, you should rate it with 4 or 5 stars. In this case, the work will be accepted as is. You can also (optionally) include a message describing your assessment of the work, which will be provided back to the crowd worker that did the work.

Last not least

You should be aware only one behavior is implemented by other programmer, while you assess the implementation just consider the behavior that the code implemented.
